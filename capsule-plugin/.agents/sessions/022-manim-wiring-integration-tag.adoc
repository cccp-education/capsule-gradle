== Réalisations

=== 1. Tag @integration sur 3 scénarios Cucumber ✅

Les 3 scénarios environnementaux (Audio constraint, Output constraint, WebM validity)
sont maintenant tagués `@integration` dans `capsule.feature`. Le `CucumberTestRunner`
filtre déjà `not @integration` et `not @wip`.

=== 2. Scénario Cucumber @manim ✅

Ajouté dans `capsule.feature` :

```gherkin
@manim
Scenario: Manim slide type is detected in capsule script and triggers ManimEngine render
  Given a reveal.js deck "manim-deck.html" with 1 slides and data-capsule-slide attributes
  And a capsule script "manim-course-script.txt" with 1 manim slide segments
  When I run the task "generateCapsuleVideo" with NoOp capture
  Then the capsule script contains a slide with type MANIM
  And the ManimEngine render is invoked for the manim slide
```

Step definitions correspondantes dans `CapsuleStepDefinitions.kt`.

=== 3. CapsuleManager.resolveManimEngine() ✅

Nouvelle méthode factory dans `CapsuleManager.companion` :

- Si `executablePath == "noop"` → `NoOpManimEngine`
- Sinon → `ManimEngineImpl(executablePath, quality)` + fallback `NoOpManimEngine` si indisponible

4 tests unitaires `CapsuleManagerResolveManimEngineTest`.

=== 4. ManimEngine wiring dans CapsuleVideoTask ✅

Ajouté dans `CapsuleVideoTask.execute()` :

- Propriété `manimEngine: ManimEngine?` injectable
- Méthode `resolveManimEngineInternal()` qui résout depuis `CapsuleExtension`
- Boucle Manim avant Playwright : pour chaque `SlideType.MANIM`, appelle `manim.render(sceneName, scriptPath, outputDir)`
- Logging des scénarios Manim détectés et du résultat

=== 5. ManimEngineWiringTest ✅

3 tests unitaires :
- `MANIM slide triggers ManimEngine render instead of Playwright`
- `MANIM slide video is produced by ManimEngine when NoOp captures`
- `MANIM slide with NoOp engine produces placeholder MP4`
