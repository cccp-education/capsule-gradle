=== 033.2a — Bug-fix: pushConfigIntoExtension DSL > merge
- **Root cause**: `CapsulePlugin.pushConfigIntoExtension()` écrasait inconditionnellement
  les propriétés DSL par les valeurs du merge (YAML/props/ENV). Le DSL `outputDir = "capsules"`
  était systématiquement remplacé par `"capsule"` (convention).
- **Fix**: `CapsuleConventions` data class snapshotée à la création de l'extension.
  `pushConfigIntoExtension` ne set que si la valeur actuelle == convention (pas de DSL explicite).
- **Impact**: 3 scénarios Cucumber `@integration` (Audio constraint, Output constraint, WebM validity)
  qui utilisaient `outputDir = "capsules"` dans le DSL sont maintenant respectés.
- **Fichiers modifiés**:
  - `CapsuleModels.kt` — ajout `CapsuleConventions` data class + `conventions` field
  - `CapsulePlugin.kt` — `pushConfigIntoExtension` avec guard convention
  - `CapsulePluginTest.kt` — 2 tests `pushConfigIntoExtension` adaptés

=== 033.2b — Bug-fix: NoOpPlaywrightCapture WebM EBML header
- **Root cause**: `NoOpPlaywrightCapture` écrivait un placeholder texte, pas un WebM binaire valide.
  Le scénario Cucumber "WebM validity constraint" vérifiait le header EBML (0x1A 0x45 0xDF 0xA3)
  et échouait car le fichier était du texte.
- **Fix**: `MINIMAL_WEBM` ByteArray avec header EBML + Segment + Info valide (hex string → chunked(2)).
  Le fichier produit est un WebM minimal valide (0 durée, 0 frame).
- **Fichiers modifiés**:
  - `PlaywrightManager.kt` — `NoOpPlaywrightCapture.capture()` → `writeBytes(MINIMAL_WEBM)`
  - `CapsulePluginTest.kt` — 4 tests adaptés (vérification header EBML au lieu de placeholder texte)
