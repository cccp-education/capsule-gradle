== Résumé

== Bugs Corrigés

=== BUG 1 — Liens reveal.js cassés dans HTML injecté (CRITIQUE) ✅

**Problème** : Le HTML injecté était écrit dans `build/capsule/injected/` mais les ressources `reveal.js/` (CSS, JS) sont dans `build/docs/asciidocRevealJs/`. Playwright ne chargeait pas le JS → Reveal undefined → vidéo statique.

**Fix** : `CapsuleManager.kt:462` — `injectedDir` pointe maintenant vers `deckFile.parentFile` (même répertoire que le deck original). Les chemins relatifs `reveal.js/` résolvent correctement.

=== BUG 2 — example-deck pollution ✅

**Problème** : `example-deck-script.txt` (template slider, 1 slide vide) était traité comme un vrai deck → génération de vidéos fantômes.

**Fix** : `CapsuleManager.kt:98-102` — `readScriptFiles()` filtre désormais les fichiers commençant par `example-` ou contenant `-context-`.

=== BUG 3 — Format string Python dans Kotlin ✅

**Problème** : `{:.1f}` est la syntaxe Python, pas SLF4J/Kotlin. Le format ne s'appliquait pas.

**Fix** : `CapsuleManager.kt:356` — Remplacé par `String.format("%.1f", value)` dans le logger.

=== BUG 4 — Injection audio sautée avec TTS noop ✅

**Problème** : `hasRealAudio` vérifiait que les fichiers MP3 n'étaient pas des placeholders (> 500 bytes, sans en-tête `# TTS PLACEHOLDER`). Avec TTS noop, tous les fichiers sont des placeholders → `hasRealAudio = false` → HTML copié sans injection audio.

**Fix** : `CapsuleManager.kt:467` — Renommé `hasRealAudio` → `hasAudio`, suppression de la vérification `!isPlaceholder()`. L'audio est injecté même avec des placeholders (permet de tester le pipeline complet).
