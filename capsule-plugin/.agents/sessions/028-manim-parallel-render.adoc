= Session 028 — Manim Parallel Render (CAP-20)

== Date
2026-06-02

== Objectif
Parallélisation ManimEngine.render() pour les slides MANIM multiples. EPIC CAP-20.

== Livrables

1. **ManimParallelRenderer interface** — `ManimParallelRenderer.kt` (143 lignes)
   - Interface avec `renderAll(slides, manimEngine, scriptsDir, outputDir): Map<Int, File>`
   - `NoOpManimParallelRenderer` — fallback séquentiel (itération simple)
   - `ManimParallelRendererImpl` — parallélisation via `Executors.newFixedThreadPool`
   - Factory `CapsuleManager.resolveManimParallelRenderer(parallelism)`
   - `parallelism <= 1` → NoOp (séquentiel), `parallelism > 1` → Impl (thread pool)

2. **14 tests unitaires** — `ManimParallelRendererTest.kt`
   - NoOp: renderAll, empty slides, skip manimScene null, skip missing script, name/available
   - Impl: NoOp engine render 3 slides, empty slides, skip null manimScene, default parallelism=4
   - Factory: parallelism=1 → NoOp, parallelism>1 → Impl

3. **CapsuleVideoTask refactor** — Boucle Manim séquentielle → `parallelRenderer.renderAll()`
   - Nouvelle propriété injectable `manimParallelRenderer`
   - Méthode `resolveManimParallelRenderer()` utilisant `manimParallelRender` de l'extension
   - Étape 1 : renderAll parallèle → `renderedFiles: Map<Int, File>`
   - Étape 2 : mux séquentiel + probe duration (dépend du render output)
   - Slide replacement utilise `renderedFiles` pour trouver les MP4 produits

4. **Scenario Cucumber @manim @parallel** — `capsule.feature` (22 scénarios total)
   - 2 nouveaux step definitions dans `CapsuleStepDefinitions.kt`

5. **Config DSL `parallelRender`** — `ManimConfig.parallelRender: Boolean = false`
   - Ajouté dans `CapsuleExtension.manimParallelRender`
   - Propagé dans `CapsuleConfig`, `CapsuleConfigLoader`, `CapsuleConfigMerger` (4 sources)
   - Propagé dans `CapsulePlugin.pushConfigIntoExtension()`
   - Propagé dans `CapsuleScaffoldTask` (YAML scaffold)
   - 8 nouveaux tests : 3 ManimConfigTest + 5 CapsuleConfigMergerTest
