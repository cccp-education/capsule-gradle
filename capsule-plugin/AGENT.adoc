= AGENT.adoc — Capsule Gradle

_Produit_ : Capsule video pedagogique comme plugin Gradle
_Licence_ : Apache 2.0 (OSS)
_DAG_ : N2 — Plugin Gradle de generation de capsules video
_Dependance_ : compileOnly slider-gradle (consomme les decks reveal.js)

== Principe

Le plugin `capsule` transforme un deck reveal.js genere par `slider-gradle`
en capsule video MP4/WebM avec TTS synchronise. Il orchestre Playwright
(pilotage + capture) et un moteur TTS (local ou API) a partir des notes
speaker deja presentes dans l'AsciiDoc source.

== Loi de l'Économie d'Encre — Non-Répétition des Opérations Coûteuses

*PRINCIPE* : Une opération coûteuse (TTS Piper/API, capture vidéo Playwright, inférence cloud) qui a déjà produit un résultat *valide et persistant* NE DOIT PAS être ré-exécutée sur la même entrée. Le résultat existant est la vérité ; le re-calcul est un gaspillage de crédits API/temps de build. Le TTS et la capture vidéo sont des services metered (API TTS) ou coûteux en CPU/temps (Playwright). Régénérer un MP3 déjà synthétisé, recapturer une vidéo dont le MP4 est valide : chacun de ces gestes consomme des ressources sans produire de nouvelle connaissance. L'économie d'encre est la condition de viabilité économique du pipeline agentique MVP0.

*RÈGLE* :
. Avant toute opération coûteuse, **vérifier l'existence d'un artefact résultat** (MP3 par slide, MP4/WebM final, script TTS généré, cache disque).
. Si l'artefact existe et couvre l'entrée demandée → **réutiliser sans ré-invoquer**.
. Seul le **delta** (slides modifiées, notes speaker éditées, audio manquant) doit être calculé — jamais l'ensemble.
. Ne **jamais écraser** un résultat validé par un re-calcul identique. L'écrasement est un symptôme de non-idempotence économique.
. L'invalidation est **explicite et ciblée** (slide éditée, note speaker modifiée), jamais globale par défaut.

*ANTI-PATTERNS SPÉCIFIQUES CAPSULE* :
. *Re-TTS systématique* : re-synthétiser les MP3 de toutes les slides parce qu'une seule note speaker a été corrigée.
. *Re-capture vidéo inconditionnelle* : relancer Playwright sur tout le deck parce qu'une slide a été ajoutée.
. *Cache-busting silencieux* : un `writeText()` qui écrase le script TTS ou le MP4 final avant même d'avoir vérifié l'existant.

*PROTECTION* :
. Lecture avant écriture : vérifier l'existence du MP3 par slide (hash du texte speaker) avant de appeler le TTS ; ne re-synthétiser que les slides dont le hash diffère.
. Delta capture : ne capturer que les slides dont l'audio ou le rendu HTML a changé depuis la dernière capture.
. `DisableCachingByDefault` ne dispense pas de l'idempotence applicative — Gradle ne sait pas ce qu'est un "MP3 déjà valide", seul le domaine le sait.

*RÉFÉRENCE* : Source canonique — `bakery-gradle/bakery-plugin/AGENT.adoc` section "Non-Répétition des Opérations Coûteuses — Loi de l'Économie d'Encre" (Session 153). Portée transverse : 10 boroughs LLM metered. Potentiel 50-80% d'économie LLM sur le pipeline MVP0 complet.

== Taches Principales

  ./gradlew generateCapsuleScript   → extraire les scripts TTS par deck
  ./gradlew generateCapsule         → generer les MP3 TTS par slide
  ./gradlew generateCapsuleVideo    → capturer la video Playwright (deck + audio)
  ./gradlew check                   → tous les tests (unitaire + fonctionnel + Cucumber)

== Tests

Le projet utilise JUnit5, GradleTestKit et Cucumber BDD.
Toute modification de code source DOIT passer `./gradlew check`
(sauf permission contraire).

== Stack

- Kotlin — DSL Gradle
- Playwright — pilotage headless Chromium + capture video
- Piper ou API TTS externe — synthese vocale a partir des notes speaker
- slider-gradle — dependance compileOnly, contrat : deck reveal.js HTML + script capsule

== Ne remplace PAS opencode

Le plugin est invoque PAR opencode (via tool use).
Il automatise la capture video, mais la validation humaine
du contenu pedagogique reste obligatoire.
